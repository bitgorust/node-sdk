# Repository Guidelines

## Project Structure & Module Organization
The SDK is authored in TypeScript from the repository root. [`index.ts`](/home/jing/Workspaces/github/node-sdk/index.ts) is the public entrypoint. Core modules live in folders such as [`client`](/home/jing/Workspaces/github/node-sdk/client), [`adaptor`](/home/jing/Workspaces/github/node-sdk/adaptor), [`dispatcher`](/home/jing/Workspaces/github/node-sdk/dispatcher), [`http`](/home/jing/Workspaces/github/node-sdk/http), [`logger`](/home/jing/Workspaces/github/node-sdk/logger), [`utils`](/home/jing/Workspaces/github/node-sdk/utils), and [`ws-client`](/home/jing/Workspaces/github/node-sdk/ws-client). API-generation helpers are under [`code-gen`](/home/jing/Workspaces/github/node-sdk/code-gen), and reference images/docs are in [`doc`](/home/jing/Workspaces/github/node-sdk/doc). Build artifacts are generated into `lib/` (CommonJS), `es/` (ESM), and `types/`; do not hand-edit those outputs.

## Build, Test, and Development Commands
Use the package scripts as the default workflow:

- `yarn build`: rebuilds `lib/`, `es/`, and `types/` with Rollup.
- `yarn test`: runs the Jest suite once with `ts-jest`.
- `yarn test:watch`: watches the focused utility tests during local iteration.

If you need linting or formatting, run `npx eslint . --ext .ts` and `npx prettier --check .`; configuration lives in `.eslintrc.js` and `.prettierrc.json`.

Repository analysis artifacts generated by local scripts should live under `doc/api-inventory/` and prefer lowercase kebab-case names, for example `doc/api-inventory/canonical.md`, `doc/api-inventory/canonical.json`, and `doc/api-inventory/raw/`.

## Coding Style & Naming Conventions
Prettier is configured for 4-space indentation, single quotes, semicolons, and trailing commas where valid in ES5. Follow existing TypeScript naming: `PascalCase` for classes and types, `camelCase` for functions and variables, and descriptive folder names by domain. The repo uses the `@node-sdk/*` path alias for internal imports. Keep public API changes centralized through the root exports in `index.ts`.

## Testing Guidelines
Tests use Jest and live in `__tests__` directories with `.ts` filenames such as `utils/__tests__/format-url.ts`. Add tests next to the module they exercise. Cover new client behavior, adapters, or websocket logic with focused unit tests before changing generated outputs. Run `yarn test` before opening a PR.

## Commit & Pull Request Guidelines
Recent history follows conventional prefixes: `feat:`, `fix:`, and `chore:`. Keep commit subjects imperative and scoped to one change, for example `fix: prevent reconnect timer leak`. PRs should explain the user-visible impact, note any API or build-output changes, link related issues, and include test evidence. Add screenshots only when updating docs or assets under `doc/`.
